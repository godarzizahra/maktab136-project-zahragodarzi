مستندات API فروشگاه

این مستندات شامل تمام endpointهای بک‌اند فروشگاه است. برای برخی APIها نیاز به Authentication Token وجود دارد.

Authentication

POST /api/auth/register

ثبت نام کاربر جدید در سیستم.

نیازی به توکن ندارد.
Body 
{
"name": "علی احمدی",
"email": "ali@example.com",
"password": "123456",
"phone": "09123456789"
}

POST /api/auth/login

کاربر با ایمیل و رمز عبور وارد سیستم می‌شود.

در پاسخ این API یک Access Token و Refresh Token دریافت می‌کنید. Access Token برای درخواست‌های بعدی استفاده می‌شود. 
{
"email": "ali@example.com",
"password": "123456"
}

POST /api/auth/refresh-token

برای گرفتن Access Token جدید زمانی که توکن قبلی منقضی شده است.

نیاز به Refresh Token دارد.
{
"refreshToken": "REFRESH_TOKEN"
}

GET /api/auth/me

اطلاعات کاربر لاگین شده را برمی‌گرداند.

نیاز به Authorization Token دارد.
Authorization: Bearer ACCESS_TOKEN

Products

GET /api/products

دریافت لیست محصولات فروشگاه.

نیازی به توکن ندارد.
Query Params (اختیاری) 
page
limit
category
search

GET /api/products/:id

دریافت اطلاعات یک محصول خاص.

نیازی به توکن ندارد.

POST /api/products

ایجاد محصول جدید در سیستم.

فقط ادمین می‌تواند از این API استفاده کند.
Authorization: Bearer ADMIN_TOKEN
Body (form-data) 
name
description
price
category
stock
brand
images (file)

PUT /api/products/:id

ویرایش اطلاعات محصول.

فقط ادمین مجاز است.

DELETE /api/products/:id

حذف یک محصول از سیستم.

فقط ادمین مجاز است.

Cart

GET /api/cart

نمایش سبد خرید کاربر.

نیاز به توکن کاربر دارد.

POST /api/cart/add

افزودن یک محصول به سبد خرید.

نیاز به توکن کاربر دارد.
{
"productId": "product_id",
"quantity": 2
}

PUT /api/cart/update/:itemId

تغییر تعداد یک محصول در سبد خرید.

نیاز به توکن کاربر دارد.
{
"quantity": 3
}

DELETE /api/cart/remove/:itemId

حذف یک آیتم از سبد خرید.

نیاز به توکن کاربر دارد.

DELETE /api/cart/clear

خالی کردن کامل سبد خرید.

نیاز به توکن کاربر دارد.

Orders

POST /api/orders

ایجاد سفارش جدید از روی سبد خرید کاربر.

نیاز به توکن کاربر دارد.
{
"shippingAddress":{
"name":"علی احمدی",
"phone":"09123456789",
"address":"تهران"
},
"paymentMethod":"cash"
}

GET /api/orders

نمایش تمام سفارش‌های کاربر.

نیاز به توکن کاربر دارد.

GET /api/orders/:id

نمایش جزئیات یک سفارش.

نیاز به توکن کاربر دارد.

PUT /api/orders/:id/status

تغییر وضعیت سفارش.

فقط ادمین مجاز است.
{
"status":"confirmed"
}

GET /api/orders/admin/all

لیست تمام سفارش‌ها برای مدیریت فروشگاه.

فقط ادمین مجاز است.
page
limit
status

Health Check

GET /api/health

برای بررسی اینکه سرور در حال اجرا است.

نیازی به توکن ندارد.
